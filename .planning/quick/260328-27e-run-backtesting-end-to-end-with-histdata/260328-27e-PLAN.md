  <action>
Create `src/fxsoqqabot/backtest/runner.py` with an async function `run_full_backtest(settings: BotSettings, config: BacktestConfig | None = None, skip_ingestion: bool = False) -> None` that orchestrates the full pipeline:

1. **Setup:** If `config` is None, create default `BacktestConfig()`. Initialize structlog logger.

2. **Step 1 - Ingestion:** Unless `skip_ingestion` is True:
   - Create `HistoricalDataLoader(config)`
   - Call `loader.ingest_all()` which reads from `config.histdata_dir` (default: `data/histdata/`)
   - Print quality report summary: files processed, total rows, duplicates removed, gaps interpolated, extreme bars removed, date range
   - Print separator line

3. **Step 2 - Walk-Forward Validation:**
   - Create `HistoricalDataLoader(config)` (fresh instance for clean DuckDB connection)
   - Create `BacktestEngine(settings, config)`
   - Create `WalkForwardValidator(engine, loader, config)`
   - Call `wf_result = await validator.run_walk_forward()`
   - Print results: number of windows, profitable %, aggregate PF, pass/fail status
   - For each window, print a single-line summary: window idx, val period, val equity, val PF, profitable yes/no
   - Print separator line

4. **Step 3 - OOS Evaluation:**
   - Call `oos_result = await validator.evaluate_oos(wf_result)`
   - Print results: OOS PF, OOS max DD, IS PF, IS max DD, PF ratio, DD ratio, pass/fail, overfit yes/no
   - Print separator line

5. **Step 4 - Monte Carlo Simulation:**
   - Collect ALL trade PnLs from all validation windows in `wf_result.windows` (use `window.val_result.trades`) into a single numpy array
   - If no trades, print warning and skip Monte Carlo
   - Otherwise call `run_monte_carlo(trade_pnls, config.starting_equity, config.n_monte_carlo, config.mc_max_drawdown_pct, config.mc_seed)`
   - Print results: simulations, 5th pct equity, median equity, 95th pct max DD, p-value, pass/fail
   - Print separator line

6. **Final Summary:** Print overall pass/fail: "PASS" only if walk-forward passes AND OOS passes AND Monte Carlo passes. Use green/red color via simple ANSI codes (\033[32m for green, \033[31m for red, \033[0m for reset).

Formatting guidelines:
- Use `print()` for all output (not structlog -- this is user-facing CLI output)
- Format floats to 2 decimal places for equity/PF, 1 decimal place for percentages
- Use simple ASCII table formatting with fixed-width columns for window results
- Header: "FXSoqqaBot Backtest Pipeline" with "=" separators
- Each section has a clear header (e.g., "[1/4] Data Ingestion", "[2/4] Walk-Forward Validation", etc.)
