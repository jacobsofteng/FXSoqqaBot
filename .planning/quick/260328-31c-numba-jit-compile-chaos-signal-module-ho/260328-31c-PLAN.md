nolds.rs(data, n, unbiased=True):
  - Reshapes data into (m, n) subsequences
  - Computes mean, cumsum, range, std per subsequence
  - Returns mean(R/S) filtering zero-range subsequences

nolds.hurst_rs(data, nvals=None, fit="RANSAC", corrected=True, unbiased=True):
  - nvals default: logmid_n(total_N, ratio=1/4.0, nsteps=15)
  - Calls rs() for each nval (15 iterations)
  - Anis-Lloyd-Peters correction via expected_rs()
  - Final RANSAC line fit on log(nvals) vs log(rsvals) - log(expected_rs)
  - Returns slope + 0.5

nolds.lyap_r(data, emb_dim=10, lag=None, min_tsep=None, tau=1, min_neighbors=20, trajectory_len=20, fit="RANSAC"):
  - FFT for auto-lag and min_tsep (NOT jittable -- stays outside @njit)
  - delay_embedding(data, emb_dim, lag)
  - O(n^2) distance matrix: [rowwise_euclidean(orbit, orbit[i]) for i in range(m)]
  - Masks temporal neighbors with inf
  - argmin for nearest neighbors
  - Divergence trajectory: mean(log(dist)) along k steps
  - RANSAC line fit (NOT jittable -- stays outside @njit)
  - Returns slope / tau

nolds.corr_dim(data, emb_dim, lag=1, rvals=None, fit="RANSAC"):
  - rvals default: logarithmic_r(0.1 * std, 0.5 * std, 1.03)
  - delay_embedding(data, emb_dim, lag)
  - O(n^2) distance matrix
  - For each r: C(r) = count(dists <= r) / (n*(n-1))
  - RANSAC line fit on log(r) vs log(C(r))
  - Returns slope

Create `src/fxsoqqabot/signals/chaos/_numba_core.py` containing all @njit(cache=True) functions. Every function takes and returns numpy arrays/scalars only -- no Python objects, no strings, no scipy, no sklearn.

**Shared utilities:**

1. `_ols_slope(x, y)` -- Simple OLS slope via manual accumulation loop. Returns float. Replaces np.polyfit inside jitted paths. Use `abs(denom) < 1e-30` guard for degenerate cases returning 0.0.

2. `_ols_slope_intercept(x, y)` -- Returns (slope, intercept) tuple. Same manual OLS but also returns intercept. Needed for lyap_r and corr_dim line fits inside jitted core.

3. `_delay_embedding(data, emb_dim, lag)` -- Time-delay embedding. Returns 2D array shape (n - (emb_dim-1)*lag, emb_dim). Nested loop filling result[i, d] = data[i + d * lag].

4. `_pairwise_euclidean(orbit)` -- O(n^2) symmetric distance matrix. Triple nested loop: i, j>i, d. Compute sqrt(sum of squared diffs). Fill dists[i,j] = dists[j,i]. Set diagonal to 0.0.

**Hurst R/S core:**

5. `_rs_single(data_slice, n, unbiased)` -- Compute R/S for a single subsequence length n. Mirrors nolds.rs():
   - total_N = len(data_slice), m = total_N // n, truncate data to m*n
   - Loop over m subsequences: compute mean, subtract mean to get departures, cumsum, range (max-min), std (with ddof=1 if unbiased)
   - Skip subsequences where range == 0
   - Return mean(R/S) or np.nan if all zero

6. `_rs_values(data, nvals, unbiased)` -- Loop over nvals array, call _rs_single for each. Returns 1D array of R/S values same length as nvals. This is the hot loop (15 iterations over full data).

**Lyapunov Rosenstein core:**

7. `_lyap_r_core(orbit, min_tsep, trajectory_len)` -- The expensive inner computation:
   - Build O(n^2) pairwise distance matrix using _pairwise_euclidean(orbit)
   - Mask temporal neighbors: for i in range(m), set dists[i, max(0, i-min_tsep):i+min_tsep+1] = np.inf
   - ntraj = m - trajectory_len + 1
   - Find nearest neighbors: for each i in range(ntraj), argmin of dists[i, :ntraj] (finite distances only)
   - Build divergence trajectory: for k in range(trajectory_len), compute mean(log(dists[i+k, nb_idx[i]+k])) over all i, skipping zeros
   - Return the div_traj array (1D, length trajectory_len). The caller does the line fit.

**Correlation dimension core:**

8. `_corr_dim_core(orbit, rvals)` -- The expensive inner computation:
   - Build O(n^2) pairwise distance matrix (upper triangle only for counting)
   - n = len(orbit)
   - For each r in rvals: count pairs where dist <= r, compute C(r) = count / (n * (n-1))
   - Return csums array (1D, same length as rvals). The caller does log transform and line fit.

**Key implementation details:**
- ALL functions decorated with `@njit(cache=True)`
- Use `np.inf` not `float("inf")`
- Use `np.empty()` / `np.zeros()` for array creation
- Use explicit loops, not np.reshape or axis operations where Numba doesn't support them
- For _rs_single: implement the reshape+mean+cumsum+range+std logic as explicit loops since np.reshape with axis operations can be tricky in Numba. Alternatively, use np.reshape and axis ops if Numba supports them (np.mean with axis is NOT supported in Numba nopython mode -- use explicit loops).
- np.mean(axis=N), np.std(axis=N), np.cumsum(axis=N), np.max(axis=N), np.min(axis=N) are NOT supported in Numba nopython mode with axis argument. Must use explicit loops over subsequences.
- np.sum, np.mean, np.std, np.max, np.min WITHOUT axis argument ARE supported.
- np.sqrt, np.log, np.abs, np.isfinite, np.isinf ARE supported.
- np.argmin without axis IS supported on 1D arrays. For 2D use explicit loop.
- For the distance matrix masking in _lyap_r_core: use np.inf assignment in a loop.
- For _corr_dim_core: only compute upper triangle distances to save half the work, then count = 2 * count_upper since matrix is symmetric (exclude diagonal which is always 0).

**Warm-up function:**

9. `warmup_jit()` -- Non-jitted Python function that calls each @njit function once with small dummy arrays (length 50) to trigger compilation. Called from ChaosRegimeModule.initialize(). This ensures first real call uses cached bytecode.
