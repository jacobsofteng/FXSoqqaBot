# Phase 6 -- UI Design Contract

> Visual and interaction contract for dashboard live state wiring. This phase fixes engine-to-dashboard data flow -- no new UI surfaces or components. All design tokens, colors, typography, and layouts inherit from Phase 4 (04-UI-SPEC.md). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

Phase 6 introduces **zero new UI surfaces**. Both dashboards (TUI and web) were built in Phase 4 and are complete. This phase fixes the engine-side wiring so that existing UI components display real data instead of defaults or broken values.

| Property | TUI (Terminal) | Web Dashboard (Browser) |
|----------|---------------|------------------------|
| Tool | Textual 8.1.1 (unchanged) | Vanilla JS (unchanged) |
| Styling | TCSS (unchanged) | Plain CSS (unchanged) |
| Component library | Textual widgets (unchanged) | lightweight-charts 2.1 + Plotly 5.x (unchanged) |
| Icon library | Rich markup symbols (unchanged) | Unicode symbols (unchanged) |
| Font | Terminal monospace (unchanged) | system-ui stack (unchanged) |

**No new libraries, no new UI files, no new CSS.** All changes are in Python engine code (`engine.py`, `state_snapshot.py`, `circuit_breakers.py`, `server.py`).

## Copywriting Contract

Phase 6 does not add new UI panels. All existing copywriting from Phase 4 (04-UI-SPEC.md) carries forward. This section documents only the **data-driven display changes** that become visible once wiring bugs are fixed.

### Newly Functional Display States

These elements already exist in the UI but currently show incorrect data. After Phase 6 fixes, they display real values:

| Element | Was Showing | Will Show After Fix |
|---------|------------|---------------------|
| TUI: Equity value | `$0.00` (default) | `$20.00` (real MT5 equity, e.g. `${equity:.2f}`) |
| TUI: Connection status | `MT5 DISCONNECTED` (always) | `REGIME: {state} ({confidence}%)` when connected |
| TUI: Kill switch status | Always truthy coroutine display | `[green]OK[/]` when not killed, `[bold red on white]KILLED[/]` when killed |
| TUI: Equity sparkline | Empty (no data points) | Rolling sparkline of last 50 equity values |
| TUI: Breaker status | Empty dict (silent error) | `Daily DD: {pct}% | Breakers: OK` or `HALTED ({reason})` |
| Web: Equity chart | Empty Plotly chart | Line chart with real equity history data points |
| Web: Summary equity | `$0.00` | `${equity}` (real value from MT5) |
| Web: Connection dot | Red dot / `Disconnected` | Green dot / `Live` when WebSocket connected |
| Web: Module weights chart | `No module weight data yet.` | Current weights: `{"chaos": 0.33, "flow": 0.33, "timing": 0.33}` (or adapted values) |
| Web: Pause button | Toggles flag but has no effect | Toggles flag AND engine loops respond to pause state |

### New State Field: `module_weights`

Added to `TradingEngineState.to_dict()` and consumed by the web dashboard:

| Element | Copy / Format |
|---------|--------------|
| Module weights key in WebSocket JSON | `"module_weights": {"chaos": 0.33, "flow": 0.33, "timing": 0.33}` |
| `/api/module-weights` response format | `{"data": [{"chaos": 0.33, "flow": 0.33, "timing": 0.33}]}` |
| Empty state (no weight tracker) | `No module weight data yet.` (existing empty state in `dashboard.js`) |

### Pause Behavior Copywriting

| Element | Copy |
|---------|------|
| TUI subtitle when paused | `[PAUSED]` (already implemented in `app.py`) |
| Web pause button (paused) | `Resume Trading` (already implemented in `dashboard.js`) |
| Web pause button (active) | `Pause Trading` (already implemented in `dashboard.js`) |

### Error and Empty States (unchanged from Phase 4)

| Element | Copy |
|---------|------|
| No trades | `No trades recorded yet. The bot will log trades as they execute.` |
| No filtered trades | `No trades match your filters.` |
| WebSocket disconnected | `Disconnected` (red dot) with auto-reconnect |
| API failure | `Failed to load data. Check that the bot is running.` |
| Kill switch activated | `All positions killed. Trading halted.` (alert dialog) |

### Destructive Actions (unchanged from Phase 4)

| Action | Trigger | Confirmation |
|--------|---------|-------------|
| Kill all positions (TUI) | Press `k` key or click Kill button | Callback fires immediately (no additional prompt beyond key binding) |
| Kill all positions (Web) | Click "Emergency Kill" button | Modal dialog: "This will close ALL positions and halt trading. Type the API key to confirm:" with password input |
| Pause trading (Web) | Click "Pause Trading" button | Prompt: "Enter API key to pause/resume trading:" |

## Data Flow Contracts (Phase 6 Specific)

These contracts define what data the engine MUST write to `TradingEngineState` for the dashboards to function correctly. This is the core deliverable of Phase 6.

### SC-1: Equity and Connection State

| Field | Source | Update Frequency | Dashboard Consumer |
|-------|--------|-----------------|-------------------|
| `equity` | `account_info.equity` from MT5 bridge | Every health loop (10s) and signal loop iteration | TUI stats panel, TUI sparkline, Web summary bar, Web equity chart |
| `is_connected` | `self._bridge.is_connected` | Every health loop (10s) and after `_connect_mt5()` | TUI regime panel (shows DISCONNECTED when false), Web connection dot |
| `_current_equity` | Engine instance attribute | Same as equity | Internal source for `_update_engine_state()` |
| `_connected` | Engine instance attribute | Same as is_connected | Internal source for `_update_engine_state()` |

### SC-2: Kill Switch Boolean

| Field | Source | Update Frequency | Dashboard Consumer |
|-------|--------|-----------------|-------------------|
| `is_killed` | `CircuitBreakerManager.is_killed` property (synchronous, reads `self._snapshot.kill_switch == BreakerState.KILLED`) | Every `_update_engine_state()` call | TUI risk panel, Web kill status |

### SC-3: Equity History and Module Weights

| Field | Source | Update Frequency | Cap | Dashboard Consumer |
|-------|--------|-----------------|-----|-------------------|
| `equity_history` | Append `s.equity` when > 0 | Every `_update_engine_state()` call | 1000 entries (trim to 500 when exceeded) | TUI sparkline (last 50), Web equity chart (all), WebSocket (last 50 via `to_dict()`) |
| `module_weights` | `self._weight_tracker.get_weights()` | Every `_update_engine_state()` call | N/A (single dict) | Web `/api/module-weights` endpoint, Web module performance chart |
| `breaker_status` | `self._breakers.get_breaker_status()` | Every `_update_engine_state()` call | N/A (6-key dict) | TUI risk panel, Web breaker display |

### SC-4: Pause Behavior

| Loop | Behavior When Paused |
|------|---------------------|
| `_signal_loop` | Skip entire body (`continue` after sleep) -- no signal evaluation, no fusion, no trade execution |
| `_tick_loop` | Skip entire body (`continue` after sleep) -- no tick polling, no paper SL/TP checking |
| `_bar_loop` | Skip entire body (`continue` after sleep) -- no bar refresh |
