    _initialize_components(): Create and wire all components:
    - MT5Bridge(exec_config)
    - StateManager(data_config.storage_path + "/state.db") with initialize()
    - MarketDataFeed(bridge, data_config)
    - TickBuffer(maxlen=data_config.tick_buffer_size)
    - BarBufferSet(data_config)

    - PaperExecutor(starting_balance) when mode=="paper"
    - OrderManager(bridge, exec_config, paper_executor)
    - SessionFilter(session_config)
    - PositionSizer(risk_config)
    - CircuitBreakerManager(risk_config, state, session)
    - KillSwitch(state, order_manager)

    _connect_mt5(): Try bridge.connect(), if fails call bridge.reconnect_loop() per D-06.

    _crash_recovery():
    - Load circuit breaker state from SQLite via breakers.load_state() per D-07
    - Check for open positions via bridge.get_positions(symbol)
    - If positions exist: close ALL via order_manager.close_all_positions() per D-05/EXEC-04
    - Check session reset via breakers.check_session_reset() per D-10
    - Set daily starting equity via breakers.set_daily_starting_equity() from account_info

    _tick_loop(): Runs while self._running:
    - Check MT5 connection via bridge.ensure_connected(), reconnect if needed
    - Fetch ticks via feed.fetch_ticks(symbol, count=100)
    - Extend tick_buffer with new ticks
    - Store ticks in storage via storage.store_ticks(ticks)
    - Check tick freshness via feed.check_tick_freshness() per Pitfall 7
    - Check spread for circuit breaker via breakers.check_spread() per D-08
    - Check paper positions SL/TP via paper_executor.check_sl_tp() if paper mode
    - Periodic flush to parquet (every 1000 polls)
    - Sleep for tick_poll_interval_ms

    _bar_loop(): Runs while self._running:
    - Fetch multi-timeframe bars via feed.fetch_multi_timeframe_bars(symbol) per DATA-03
    - Update bar_buffers for each timeframe
    - Sleep for bar_refresh_interval_seconds

    _health_loop(): Runs while self._running (every 10s):
    - Check session reset per D-10
    - Get account info, check equity for circuit breakers
    - Save account snapshot
    - Log tripped breakers

    start():
    - Setup logging
    - Initialize components
    - Connect to MT5 (abort if cannot connect)
    - Run crash recovery
    - Set self._running = True
    - asyncio.gather(_tick_loop(), _bar_loop(), _health_loop())

    stop():
    - Set self._running = False
    - Flush storage to parquet, close storage
    - Close state DB
    - Shutdown MT5 bridge

    Properties: is_running, kill_switch, breakers (for CLI and Phase 2 access)

    Tests: Mock all components. Test:
    - _crash_recovery() calls close_all_positions when positions exist
    - _crash_recovery() calls load_state
    - _initialize_components() creates all components
    - stop() calls shutdown on bridge and close on state/storage
