<objective>
Fix trade logging pipeline so that every trade open and close is captured in DuckDB trade_log, unblocking LEARN-01/02/03/05.

Purpose: The trade_log table is the data source for GA evolution fitness, ML classifier training, signal combination analysis, and the web dashboard /api/trades endpoint. Currently it is always empty because: (1) log_trade_open is guarded by `hasattr(decision, 'fill')` which is always False since TradeDecision has no fill field, (2) log_trade_close is never called on SL/TP triggers, and (3) on_trade_closed is never called to feed the learning loop.

Output: Working trade logging pipeline where every buy/sell execution logs an open row and every SL/TP close updates the row and triggers the learning loop.
</objective>

       ```python

               # PnL from the paper executor's close
               contract_size = 100.0
               if pos.action == "buy":
                   pnl = (close_fill.fill_price - pos.open_price) * pos.volume * contract_size
               else:
                   pnl = (pos.open_price - close_fill.fill_price) * pos.volume * contract_size

               # Hold duration
               hold_duration = (datetime.now(UTC) - pos.open_time).total_seconds()

               # Current regime for exit_regime
               exit_regime = "unknown"
               if self._last_signals:

               # Log trade close
               if self._trade_logger:
                   try:
                       self._trade_logger.log_trade_close(
                           ticket=ticket,
                           exit_price=close_fill.fill_price,
                           pnl=pnl,
                           hold_duration_seconds=hold_duration,
                           exit_regime=exit_regime,

                   except Exception:

               # Clear trade manager position state
               if self._trade_manager:
       ```
