# Phase 4 -- UI Design Contract

> Visual and interaction contract for two UI surfaces: Textual TUI (terminal) and FastAPI web dashboard (browser). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

This phase has **two distinct UI surfaces** with separate tooling:

| Property | TUI (Terminal) | Web Dashboard (Browser) |
|----------|---------------|------------------------|
| Tool | Textual 8.1.1 | Vanilla JS (no framework) |
| Styling | TCSS (Textual CSS) | Plain CSS |
| Component library | Textual widgets (DataTable, Sparkline, Static, Button, RichLog) | lightweight-charts 2.1 + Plotly 5.x |
| Icon library | Rich markup emoji/symbols | Unicode symbols (no icon library) |
| Font | Terminal monospace (inherits from user shell) | `system-ui, -apple-system, "Segoe UI", sans-serif` (system stack) |
| Color rendering | Rich Text ANSI colors via `[bold green]` markup | CSS custom properties (hex values) |

**No shadcn, no React, no npm.** Both surfaces are lightweight: the TUI is pure Python/Textual, the web dashboard is a single HTML page with inline JS served by FastAPI.

## Spacing Scale

### TUI Spacing (Textual CSS units -- terminal cells)

Textual CSS uses integer cell units, not pixels. All values are in terminal character cells.

| Token | Value | Usage |
|-------|-------|-------|
| tight | 0 | No gap between adjacent panels |
| xs | 1 | Panel internal padding, inline element gaps |
| sm | 1 | Margin between adjacent panels |
| md | 2 | Section padding within panels |
| lg | 3 | Major section breaks (top/bottom bar gaps) |

Exceptions: Kill switch button gets `dock: bottom` with 0 margin (always visible at panel bottom).

### Web Dashboard Spacing (CSS pixels, 8-point scale)

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xs | `--space-xs` | 4px | Icon gaps, inline label padding |
| sm | `--space-sm` | 8px | Compact card internal padding |
| md | `--space-md` | 16px | Default card padding, element spacing |
| lg | `--space-lg` | 24px | Section padding between chart panels |
| xl | `--space-xl` | 32px | Page margin, major layout gaps |
| 2xl | `--space-2xl` | 48px | Page header/footer spacing |

Exceptions: Chart containers get 0 internal padding (charts render edge-to-edge within their card).

## Typography

### TUI Typography

Terminal renders in monospace at whatever size the user's terminal is configured to. No font control. Emphasis is achieved through Rich markup:

| Role | Markup | Weight | Usage |
|------|--------|--------|-------|
| Body | plain text | normal | Default panel content, log lines |
| Label | `[dim]text[/]` | dim | Secondary info, timestamps, units |
| Emphasis | `[bold]text[/]` | bold | Panel titles, active values, signal names |
| Alert | `[bold red]text[/]` or `[bold green]text[/]` | bold+color | Regime state, P&L values, warnings |

### Web Dashboard Typography

| Role | Size | Weight | Line Height | CSS Variable |
|------|------|--------|-------------|-------------|
| Body | 14px | 400 (regular) | 1.5 | `--font-body` |
| Label | 12px | 500 (medium) | 1.4 | `--font-label` |
| Heading | 18px | 600 (semibold) | 1.2 | `--font-heading` |
| Display | 28px | 700 (bold) | 1.1 | `--font-display` |

Font family: `system-ui, -apple-system, "Segoe UI", Roboto, sans-serif` for all roles.

Monospace for numerical values (prices, P&L, lot sizes): `"Cascadia Code", "Fira Code", "JetBrains Mono", monospace`.

## Color

### TUI Color Scheme (Rich ANSI colors)

Textual uses named ANSI colors. The terminal's theme controls exact rendering.

| Role | Rich Color | Usage |
|------|-----------|-------|
| Background | terminal default (dark) | App background, inherits terminal |
| Panel border | `$accent` (Textual CSS default) | All `.panel` borders |
| Regime: trending (favorable) | `green` / `bold green` | Regime status text, favorable signal bars |
| Regime: ranging (neutral) | `yellow` / `bold yellow` | Regime status text, neutral signal bars |
| Regime: high-chaos / pre-bifurcation (danger) | `red` / `bold red` | Regime status text, danger indicators |
| Positive P&L | `green` | Profit values, winning trades |
| Negative P&L | `red` | Loss values, losing trades |
| Kill switch button | `variant="error"` (Textual red) | Kill button background |
| Mutation event highlight | `bold magenta` | Highlighted rows for strategy mutation events (D-03) |
| Dim/secondary text | `dim` | Timestamps, labels, units |
| Confidence bars | `green` (>0.7), `yellow` (0.4-0.7), `red` (<0.4) | Per-module signal confidence visualization |

**Traffic-light regime mapping (D-02):**

| Regime State | Color | Meaning |
|-------------|-------|---------|
| trending_up | green | Favorable -- trade actively |
| trending_down | green | Favorable -- trade actively (short side) |
| ranging | yellow | Neutral -- selective trading |
| high_chaos | red | Danger -- reduce activity |
| pre_bifurcation | red | Danger -- reduce activity |

### Web Dashboard Color Scheme (CSS hex values)

Dark theme to match trading terminal aesthetic and reduce eye strain during extended monitoring.

| Role | Hex | CSS Variable | Usage |
|------|-----|-------------|-------|
| Dominant (60%) | `#0f1117` | `--color-bg` | Page background |
| Surface | `#1a1d27` | `--color-surface` | Card/panel backgrounds |
| Secondary (30%) | `#252836` | `--color-secondary` | Borders, dividers, table header bg |
| Text primary | `#e1e4eb` | `--color-text` | Body text, values |
| Text secondary | `#8b8fa3` | `--color-text-dim` | Labels, timestamps, units |
| Accent (10%) | `#3b82f6` | `--color-accent` | Active tab indicator, selected filter pill, chart crosshair |
| Positive | `#22c55e` | `--color-positive` | Profit values, winning trade markers, equity curve up-segments |
| Negative | `#ef4444` | `--color-negative` | Loss values, losing trade markers, drawdown overlay, equity curve down-segments |
| Warning | `#eab308` | `--color-warning` | Ranging regime, medium confidence, pause button |
| Destructive | `#dc2626` | `--color-destructive` | Kill switch button background, circuit breaker active |

**Accent reserved for:** active navigation tab underline, selected filter pills in trade history, chart crosshair highlight, WebSocket connection status indicator.

**Regime colors in web (same traffic-light as TUI):**

| Regime | Color Variable | Hex |
|--------|---------------|-----|
| trending_up / trending_down | `--color-positive` | `#22c55e` |
| ranging | `--color-warning` | `#eab308` |
| high_chaos / pre_bifurcation | `--color-negative` | `#ef4444` |

## TUI Panel Layout Contract

Single-screen, three-column layout per D-01. All panels visible simultaneously without scrolling.

```
+----------------------------------------------------------+
| Header: FXSoqqaBot - XAUUSD Scalper                     |
+------------------+-------------------+-------------------+
| REGIME           | OPEN POSITION     | TODAY             |
| TRENDING_UP (82%)| BUY 0.01 @ 2345  | Trades: 3         |
|                  | P&L: +$1.20      | Win: 67%          |
+------------------+ SL: 2340.00      | P&L: +$2.10       |
| SIGNAL CONFIDENCE|                   | Equity: $22.10    |
| Chaos:  ^ 74%   +-------------------+-------------------+
| Flow:   ^ 68%   | RISK STATUS       | [Equity Sparkline]|
| Timing: v 42%   | Daily DD: 1.2%/5% |                   |
+------------------| Breakers: OK      +-------------------+
| ORDER FLOW       |                   |                   |
| Delta: +142      +-------------------+ [KILL ALL         |
| Bid: 62% Ask: 38%| RECENT TRADES    |  POSITIONS]       |
+------------------| Time  Act  P&L   |                   |
                   | 14:32 BUY +$0.80 |                   |
                   | 14:15 SELL -$0.20|                   |

                   |  1.5 -> 1.8      |                   |

+----------------------------------------------------------+
```

**Column widths:** `1fr | 1fr | 1fr` (equal thirds).

**Panel heights:**
- Regime panel: 3 cells (fixed)
- Signals panel: auto (fits 3 module rows)
- Order flow panel: 3 cells (fixed)
- Position panel: 5 cells (fixed)
- Risk panel: 3 cells (fixed)
- Trades table: `1fr` (fills remaining vertical space, scrollable)
- Stats panel: 5 cells (fixed)
- Equity sparkline: 4 cells (fixed)
- Kill button: `dock: bottom` in right column

## Interaction Contracts

### TUI Interactions

| Action | Trigger | Behavior |
|--------|---------|----------|
| Kill all positions | Press `k` key OR click Kill button | Show confirmation prompt. On Enter: activate kill switch, display `[KILLED]` status. On Escape: cancel. |
| Pause trading | Press `p` key | Toggle pause. Display `[PAUSED]` in header. Bot stops opening new trades but manages existing positions. |
| Quit TUI | Press `q` key | Confirm quit. TUI exits but trading engine continues running headless. |
| Scroll trades | Arrow keys in trades table | Standard DataTable cursor navigation within the recent trades panel only. |
| Refresh | Automatic every 1 second (D-05) | All panels refresh from shared TradingEngineState. Trades panel updates on events only. |

### Web Dashboard Interactions

| Action | Trigger | Behavior |
|--------|---------|----------|
| Kill all positions | Click "Emergency Kill" button | Modal dialog with API key input field. Submit sends POST `/api/kill?api_key={key}`. Success: display confirmation banner. Failure: display error. |
| Pause/resume trading | Click "Pause Trading" / "Resume Trading" | POST `/api/pause` or `/api/resume` with API key. Button text toggles. |
| Filter trades | Select filter controls | GET `/api/trades?regime={}&outcome={}&start_date={}&end_date={}&min_confidence={}`. Table reloads with filtered results. |
| Switch tabs | Click tab label | Show/hide tab content panels. URL does not change (single page). |
| Chart zoom/pan | Mouse scroll / drag on chart | Handled natively by lightweight-charts and Plotly. |
| Chart crosshair | Mouse hover on chart | lightweight-charts shows price/time crosshair. Plotly shows hover tooltip. |
| WebSocket reconnect | Connection lost | Auto-reconnect with exponential backoff (1s, 2s, 4s, 8s max). Display "Disconnected" status. |

## Registry Safety

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|
| shadcn official | none | not applicable -- Python project |
| Third-party | none | not applicable |
| npm / CDN | TradingView Lightweight Charts JS (CDN in HTML) | Pinned version URL, SRI hash required |
| PyPI | textual, fastapi, uvicorn, plotly, deap, scikit-learn, optuna, lightweight-charts | Versions pinned in pyproject.toml, installed via uv lockfile |

**CDN dependency:** The web dashboard includes TradingView's Lightweight Charts JS library via CDN script tag. Pin to a specific version and include a subresource integrity (SRI) hash:

```html
<script src="https://unpkg.com/lightweight-charts@4.1.3/dist/lightweight-charts.standalone.production.js"
        integrity="{sha384-hash}"
        crossorigin="anonymous"></script>
```

Plotly is served via its Python-generated JSON + Plotly.js CDN:

```html
<script src="https://cdn.plot.ly/plotly-2.35.0.min.js"
        integrity="{sha384-hash}"
        crossorigin="anonymous"></script>
```

**Alternative to CDN:** Bundle the JS files in `dashboard/web/static/vendor/` and serve from FastAPI. Eliminates external network dependency. Recommended for production since the bot runs offline on a trading machine.
