<objective>
Add walk-forward validation as a mandatory second promotion gate for shadow variants, satisfying LEARN-06.

Purpose: Without this gate, shadow variants can be promoted to live trading purely on statistical significance of paper trading results (Mann-Whitney U test). This violates the scientific validation requirement (D-18): "Walk-forward validation on recent windows must also pass." The WalkForwardValidator class exists in backtest/validation.py but is never called during the promotion path.

Output: Dual-gate promotion where a variant must pass BOTH statistical significance (p < alpha) AND a walk-forward validation check before being promoted to live. OBS requirements are carried through because the existing dashboard artifacts are already verified and need no modification.
</objective>

    **Design Decision: Lightweight Walk-Forward Gate**

    The full WalkForwardValidator requires BacktestEngine + HistoricalDataLoader + BacktestConfig and runs full backtests (potentially minutes). This is too heavy for the promotion hot path (called after every trade close). Instead:

    1. Add a `validate_variant` callback protocol to LearningLoopManager that can be injected by the engine.
    2. The engine provides a callback that runs a simplified walk-forward check using the existing infrastructure.
    3. If no callback is provided, the gate is skipped with a warning (graceful degradation).

       # Callable that takes variant params and returns bool (pass/fail)
       # Injected by engine or set to None for statistical-only mode

    3. Add a setter method:

       def set_walk_forward_validator(
           self, validator: Callable[[dict[str, float]], bool]
       ) -> None:
           """Set the walk-forward validation callback for LEARN-06.

           The callback receives a dict of variant params (param_name -> value)
           and returns True if walk-forward validation passes, False otherwise.
           Called only when a variant passes the Mann-Whitney statistical gate.

           Args:
               validator: Callable that takes param dict and returns bool.
           """

    4. Modify `_check_promotions` to add the second gate AFTER statistical significance check but BEFORE calling `promote_variant`:

       ```python
       def _check_promotions(self) -> list[dict]:
           mutations: list[dict] = []
           live_trades = self._trade_logger.query_trades(limit=200)
           variants = self._shadow.get_variants()

           for variant in variants:
               eval_result = self._shadow.evaluate_promotion(
                   variant, live_trades, self._current_equity
               )

               if eval_result.get("should_promote", False):
                   # LEARN-06: Walk-forward validation gate
                   wf_pass = True  # Default if no validator
                   if self._walk_forward_validator is not None:
                       try:
                           wf_pass = self._walk_forward_validator(
                               variant.mutated_params
                           )
                           logger.info(
                               "walk_forward_gate_result",
                               variant_id=variant.variant_id,
                               walk_forward_pass=wf_pass,
                           )
                       except Exception:
                           logger.error(
                               "walk_forward_gate_error",
                               variant_id=variant.variant_id,
                               exc_info=True,
                           )
                           wf_pass = False  # Fail-safe: reject on error
                   else:
                       logger.warning(
                           "walk_forward_validator_not_set",
                           variant_id=variant.variant_id,
                           msg="Promoting on statistical significance only -- LEARN-06 gate inactive",
                       )

                   if not wf_pass:
                       logger.info(
                           "variant_promotion_rejected_walk_forward",
                           variant_id=variant.variant_id,
                           p_value=eval_result.get("p_value"),
                       )
                       # Reset variant since it passed stats but failed walk-forward
                       self._shadow.reset_variant(variant.variant_id)
                       continue

                   # Both gates passed -- promote
                   promoted_params = self._shadow.promote_variant(
                       variant.variant_id
                   )

                   mutation_event = {
                       "mutation": True,
                       "param": "multiple",
                       "old": "live_params",
                       "new": str(promoted_params),
                       "reason": f"variant promoted p={eval_result.get('p_value', 0.0):.4f}, walk-forward={'pass' if wf_pass else 'fail'}",
                   }
                   mutations.append(mutation_event)
                   self._last_mutation_events.append(mutation_event)

                   logger.info(
                       "variant_promoted_to_live",
                       variant_id=variant.variant_id,
                       p_value=eval_result.get("p_value"),
                       walk_forward_pass=wf_pass,
                   )

           return mutations
       ```
